# 📸 Кластеризация лиц - FastAPI версия

Веб-приложение для автоматической кластеризации лиц на фотографиях и распределения изображений по группам с помощью машинного обучения.

## 🔧 Версия 1.2.0 - GCN-based Кластеризация

**✅ Добавлен революционный GCN-based алгоритм кластеризации лиц!**

- **🧠 GCN (Graph Convolutional Networks)** для link prediction
- **⚡ FAISS** для быстрого построения kNN графов
- **🔗 Adaptive edge selection** с mutual kNN и квантильными порогами
- **🎯 Link prediction** для точного определения связей между лицами
- **🕸️ NetworkX** для анализа компонент связности
- **🔄 Автоматический fallback** на традиционный алгоритм при отсутствии зависимостей
- **📊 Расширенный API статуса** с информацией о доступности GCN

**Теперь доступны два алгоритма: GCN-based (приоритет) и традиционный (fallback)!**

## 📋 API Endpoints

### Навигация
- `GET /api/drives` - список дисков и спецпапок
- `GET /api/folder?path={path}` - содержимое папки

### Загрузка файлов
- `POST /api/upload?path={path}` - загрузка файлов в папку

### Очередь обработки
- `GET /api/queue` - получить очередь
- `POST /api/queue/add` - добавить папку в очередь
- `DELETE /api/queue` - очистить очередь

### Обработка
- `POST /api/process` - запустить обработку очереди
- `GET /api/tasks` - статус всех задач
- `GET /api/tasks/{task_id}` - статус конкретной задачи

### Превью
- `GET /api/image/preview?path={path}&size={size}` - превью изображения

## ⚙️ Алгоритм работы

### GCN-based Кластеризация лиц (Приоритетный)
1. **Детекция лиц** - InsightFace Buffalo_L модель
2. **Извлечение эмбеддингов** - L2-нормированные векторы признаков
3. **Построение kNN графа** - FAISS для быстрого поиска соседей
4. **Adaptive edge selection** - mutual kNN с квантильными порогами
5. **GCN обучение** - link prediction для определения связей
6. **Кластеризация** - компоненты связности в финальном графе

### Традиционная Кластеризация (Fallback)
1. **Детекция лиц** - InsightFace Buffalo_L модель
2. **Извлечение эмбеддингов** - нормированные векторы признаков
3. **Кластеризация** - HDBSCAN по косинусным расстояниям
4. **Фильтрация** - минимальный размер кластера и уверенность детекции

### Распределение файлов
- **Один кластер** → файл перемещается в папку `{cluster_id}/`
- **Несколько кластеров** → файл копируется в каждую папку, оригинал удаляется
- **Автоочистка** пустых исходных папок
- **Исключение** папок с названием "общие"

## 🔧 Конфигурация

### GCN-based Алгоритм (Приоритетный)
```python
# Размер детекции
det_size=(640, 640)

# Минимальная уверенность детекции лица
min_score=0.5

# Параметры kNN графа
K=50                    # Количество ближайших соседей
min_k=5                 # Минимальное количество рёбер на узел
max_k=25                # Максимальное количество рёбер на узел
q=0.75                  # Квантильный порог для отбора рёбер

# Параметры GCN
epochs=50               # Количество эпох обучения
threshold=0.5           # Порог вероятности для финальной кластеризации

# Провайдеры ONNX (CPU/GPU)
providers=("CPUExecutionProvider",)
```

### Традиционный Алгоритм (Fallback)
```python
# Размер детекции (больше = точнее, но медленнее)
det_size=(1024, 1024)

# Минимальная уверенность детекции лица
min_score=0.5  

# Минимальный размер кластера
min_cluster_size=2

# Минимальное количество соседей для HDBSCAN
min_samples=1

# Провайдеры ONNX (CPU/GPU)
providers=("CPUExecutionProvider",)
```

## 📈 Производительность

### GCN-based Алгоритм
- **Скорость**: ~50-200 изображений/минуту (зависит от размера и железа)
- **Точность**: очень высокая благодаря link prediction
- **Масштабируемость**: обработка тысяч фотографий
- **Память**: требует больше RAM для GCN обучения

### Традиционный Алгоритм
- **Скорость**: ~100-500 изображений/минуту (зависит от размера и железа)
- **Точность**: высокая благодаря Buffalo_L модели
- **Масштабируемость**: обработка десятков тысяч фотографий
- **Память**: автоматическая оптимизация для больших наборов

## ❗ Известные ограничения

### Общие ограничения
- Модель InsightFace загружается при первом запуске (~100MB)
- Большие изображения обрабатываются дольше
- Требуется хорошее освещение лиц на фото
- Очень похожие лица могут попасть в один кластер

### GCN-based алгоритм
- Требует больше RAM для обучения GCN
- Медленнее традиционного алгоритма
- Зависит от качества kNN графа
- Может быть избыточным для простых случаев

### Традиционный алгоритм
- Менее точный для сложных случаев
- Может создавать слишком много мелких кластеров
- Зависит от параметров HDBSCAN

5. При проблемах с GCN - проверьте установку PyTorch и PyTorch Geometric
